# Admin Web R7.1 — Stock profundo (Implementation Plan)

**Goal:** Convertir el cascarón de Stock en módulo funcional. Override engine completo con estado computed, drops calendario interactivo, conflict detection entre drops, smart filters avanzados, bulk drops, templates reutilizables, integración con Comercial (sync de drops a campañas).

**Pre-requisito:** R7 mergeado y deployed (skeleton + Vault profundo).

**Branch:** `admin-web-r7.1`

**Estimación:** 5-7 días (~40h)

---

## Scope

### Incluye
- Drops Tab con cards funcional + smart filters reactivos
- Calendario Tab con timeline visual interactivo (drag-drop fechas)
- Universo Tab con tabla densa + filters específicos de Stock
- Override engine completo: status computed real-time vía SQL view
- Drop conflict detector: una jersey con múltiples drops simultáneos solapados
- Templates de drop reutilizables ("Drop estándar 7 días Q475 con badge X")
- Integración con Comercial: cuando drop arranca → notify Comercial Inbox para sync de campaña
- Webhook al worker cuando drop changes status (LIVE → ENDED) para invalidar cache CDN
- Bulk drop creator: select N jerseys del Vault Universo → "Crear drop conjunto"

### NO incluye (queda para R7.5+)
- A/B testing de drops (versiones de copy/badge)
- Drop pricing dinámico basado en stock real (cruza con Inventario)
- ML-based drop optimization (mejor fecha/precio)
- Multi-currency display

### Schema delta (incremental sobre R7)

```sql
CREATE TABLE IF NOT EXISTS drop_templates (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  product       TEXT NOT NULL CHECK (product IN ('stock', 'mystery')),
  slug          TEXT NOT NULL UNIQUE,
  display_name  TEXT NOT NULL,
  config        TEXT NOT NULL,      -- JSON: duration_days, price_override, badge, copy_template, priority
  created_at    INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE TABLE IF NOT EXISTS drop_conflicts (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  product         TEXT NOT NULL CHECK (product IN ('stock', 'mystery')),
  family_id       TEXT NOT NULL,
  override_a_id   INTEGER NOT NULL,
  override_b_id   INTEGER NOT NULL,
  conflict_type   TEXT NOT NULL,    -- 'overlap_dates', 'duplicate_active', 'price_inconsistency'
  detected_at     INTEGER NOT NULL DEFAULT (unixepoch()),
  resolved_at     INTEGER
);

CREATE INDEX IF NOT EXISTS idx_drop_conflicts_unresolved ON drop_conflicts(family_id) WHERE resolved_at IS NULL;
```

- [ ] Pseudo-código:
```python
def detect_drop_conflicts():
    overrides = db.execute("SELECT * FROM stock_overrides WHERE status IN ('scheduled', 'live')")
    grouped = group_by(overrides, 'family_id')

    for family_id, drops in grouped.items():
        if len(drops) < 2: continue
        for a, b in combinations(drops, 2):
            if dates_overlap(a, b):
                insert_conflict(family_id, a.id, b.id, 'overlap_dates')
            if a.price_override != b.price_override and dates_overlap(a, b):
                insert_conflict(family_id, a.id, b.id, 'price_inconsistency')
```
- [ ] Schedule cron `0 */2 * * *` (cada 2h)
- [ ] Auto-resolve cuando alguno se elimina/extiende sin overlap
